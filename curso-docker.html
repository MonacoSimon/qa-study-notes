1. Introducción a la Contenedorización

Docker es una plataforma de código abierto diseñada para empaquetar, distribuir y ejecutar aplicaciones dentro de entornos aislados denominados contenedores. Al encapsular el software junto con todas sus dependencias (librerías, archivos de configuración y binarios), se garantiza el comportamiento idéntico del sistema sin importar si se ejecuta en la computadora local de un desarrollador, en un servidor de pruebas o en la infraestructura de la nube.

2. Contenedores frente a Máquinas Virtuales

Es común confundir el software de contenedores con la virtualización tradicional, pero sus arquitecturas internas operan de manera muy distinta:

  • Máquinas Virtuales (VM): Requieren un hipervisor para simular hardware completo. Cada VM incluye un sistema operativo invitado completo, lo que las vuelve pesadas, lentas al iniciar y demandantes en memoria RAM y disco.
  • Contenedores Docker: Comparten el núcleo (kernel) del sistema operativo anfitrión. No clonan el hardware, lo que los convierte en procesos extremadamente ligeros, con arranques instantáneos de pocos segundos y un consumo mínimo de recursos indispensables.

3. Conceptos Clave de Docker

Para dominar el ecosistema, se debe asimilar la analogía de la gastronomía o la construcción:

  • Dockerfile: Archivo de texto plano con instrucciones secuenciales que actúan como la receta de cocina o el plano técnico para construir una imagen.
  • Imagen (Image): Es el paquete estático, inmutable y empaquetado resultante de compilar un Dockerfile (el equivalente a la receta preparada o molde listo). Contiene el sistema base (como Ubuntu o Alpine), configuraciones y el código de la app.
  • Contenedor (Container): Es la instancia viva, dinámica y en ejecución de una imagen (siguiendo la analogía: el plato de comida servido en la mesa).
  • Volumen (Volume): Mecanismo de almacenamiento persistente mapeado fuera del ciclo de vida del contenedor. Dado que los contenedores son efímeros por naturaleza (si se destruyen, sus datos se borran), los volúmenes salvan la información de forma permanente.
  • Red (Network): Canal virtual aislado que permite la comunicación interna y segura entre múltiples contenedores.

4. Instalación y Verificación del Entorno

Nota: Para entornos Linux (Ubuntu/Debian), es indispensable que la virtualización esté habilitada en la BIOS del procesador antes de iniciar.

# Actualizar el índice de paquetes locales
sudo apt update && sudo apt upgrade -y

# Instalar dependencias previas de red y seguridad
sudo apt install apt-transport-https ca-certificates curl software-properties-common -y

# Agregar la clave criptográfica oficial de Docker
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

# Instalar el motor de Docker y su interfaz de comandos
sudo apt install docker-ce docker-ce-cli containerd.io -y

# Añadir tu usuario al grupo docker para evitar usar 'sudo' en cada instrucción
sudo usermod -aG docker $USER
newgrp docker

Resolución de Errores de Arranque

Si al ejecutar docker run hello-world se muestra el mensaje "Cannot connect to the Docker daemon...", significa que el motor está apagado. Se soluciona forzando el inicio del servicio del sistema:

sudo systemctl start docker

5. Comandos Esenciales de la CLI

La línea de comandos permite inspeccionar el estado del motor y administrar los ciclos de vida de los componentes:

Comando Acción / Propósito
docker --version Muestra la versión de Docker instalada en el sistema.
docker images Lista todas las imágenes descargadas o compiladas localmente.
docker ps Lista únicamente los contenedores que se están ejecutando activamente.
docker ps -a Muestra el historial completo de contenedores (activos y detenidos).
docker start [ID_O_NOMBRE] Vuelve a encender un contenedor que se encontraba apagado.
docker stop [ID_O_NOMBRE] Apaga de manera ordenada un contenedor en ejecución.
docker logs [ID] Muestra las salidas de consola e historial impreso por el contenedor.
docker logs -f [ID] Inspecciona los logs en tiempo real manteniendo la consola abierta (Follow).
docker exec -it [ID] sh Abre una terminal interactiva (shell) dentro de un contenedor activo.

6. Anatomía y Construcción de un Dockerfile

Las distribuciones ligeras como Alpine Linux son ampliamente utilizadas en entornos profesionales debido a que pesan pocos megabytes, reduciendo drásticamente el tamaño final de la imagen.

Ejemplo Práctico de Dockerfile Pro

# 1. Imagen base oficial ligera
FROM alpine:latest

# 2. Establecer el directorio de trabajo interno (creará la carpeta si no existe)
WORKDIR /app

# 3. Copiar archivos locales al directorio del contenedor
COPY ./html/index.html /app/
COPY ./html/script.sh /app/

# 4. Ejecutar comandos de infraestructura en la construcción (Instalar Nginx)
RUN apk add --no-cache nginx

# 5. Otorgar permisos de ejecución al script operativo
RUN chmod +x /app/script.sh

# 6. Documentar el puerto de red expuesto internamente
EXPOSE 80

# 7. Comando definitivo por defecto al iniciar el contenedor
CMD ["/app/script.sh"]

7. Orquestación Multicontenedor con Docker Compose

Docker Compose es una herramienta que permite definir y correr aplicaciones de múltiples contenedores utilizando un único archivo de configuración estructurado en formato YAML (docker-compose.yml).

Ejemplo Práctico de Orquestación (Servidor Web + Contenedor de Soporte)

version: '3.8'

services:
  web-server:
    image: nginx:alpine
    container_name: servidor_web_frontend
    ports:
      - "8080:80"
    volumes:
      - ./html:/usr/share/nginx/html
    networks:
      - red-aplicacion

  backend-task:
    image: alpine:latest
    container_name: contenedor_soporte_alpine
    volumes:
      - ./html:/html
    command: sh -c "while true; do sleep 1000; done"
    networks:
      - red-aplicacion

networks:
  red-aplicacion:
    driver: bridge

Comandos Clave de Docker Compose

  • Encender todos los servicios del archivo: Levanta la infraestructura en segundo plano (detached mode). 
    docker compose up -d
  • Apagar y destruir los servicios: Detiene los contenedores y limpia las redes virtuales creadas. 
    docker compose down
